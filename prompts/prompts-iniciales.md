## Objetivo
* Convertir el **ERD** (Mermaid) a DDL SQL Postgres con buenas prácticas (constraints, índices, normalización mínima, timestamps).
* Traducir el diseño a Prisma (schema.prisma) y crear migraciones coherentes.
* Añadir seed de datos realistas para pruebas (empresas, posiciones, flujos, candidatos, aplicaciones, entrevistas).
* Proveer consultas de verificación (SQL y Prisma) y pasos para PGAdmin.
* Suposiciones/Naming
* Base de datos: PostgreSQL.
* snake_case en BD; camelCase en Prisma/TS.
* IDs INT GENERATED BY DEFAULT AS IDENTITY (coherente con el ERD). Si el proyecto exige UUID, propón variante.
* Campos comunes: created_at timestamptz NOT NULL DEFAULT now(), updated_at timestamptz NOT NULL DEFAULT now().
status y employment_type como TEXT + CHECK o tablas de catálogos; mantén INTERVIEW_TYPE como tabla (está en el ERD).
FKs con ON DELETE RESTRICT por defecto; usar CASCADE cuando sea claramente relacional derivado (p. ej., borrar COMPANY elimina EMPLOYEE, POSITION).

Validaciones: salary_min <= salary_max, order_index > 0, application_deadline >= current_date (opcional soft check).
Timestamps created_at, updated_at (trigger para updated_at con now()).

onDelete: Cascade donde aplique (p. ej., borrar POSITION elimina sus APPLICATION e INTERVIEW).
